# Пост 2: CLAUDE.md

**Из чего состоит CLAUDE.md на 768 строк**

CLAUDE.md — файл в корне проекта, который Claude Code читает при каждом запуске.

Без него агент — это новичок в первый день. С ним — человек, прошедший онбординг. Знает архитектуру, знает правила, знает куда не лезть.

По сути CLAUDE.md — самый ценный файл в проекте. Он содержит всё, что нужно для эффективной работы с кодовой базой. Не только для AI — для любого нового участника.

У меня в production-проекте (7 сервисов, 2 базы, Redis, Telegram бот) файл вырос до 768 строк и 19 секций. Вот какие и зачем:

**Обязательные (без них агент будет ошибаться):**

— Что за проект — 3 строки, контекст за секунду
— Архитектура — ASCII-схема сервисов и связей. Агент отлично парсит ASCII
— Деплой и запреты — "НИКОГДА не деплоить через docker run". Без этого агент "поможет" и сломает CI
— Схема БД — таблицы, поля. Иначе угадывает при каждом запросе

**Полезные (ускоряют работу):**

— Команды диагностики — SSH, docker logs, health checks
— API примеры — готовые curl-запросы
— Аутентификация — роли, тиры, cookie

**Продвинутые (для сложных систем):**

— Redis Events — каналы, кто публикует, кто слушает
— Shared модуль — откуда импортировать, что нельзя дублировать
— Troubleshooting — рецепты для типовых проблем
— Known Limitations — чтобы агент не бился головой о стену

Каждая секция появилась потому что агент без неё ошибался. Это не документация ради документации — это онбординг-гайд, который работает 24/7.

**С чего начать:**

Набери /init в Claude Code. Он сам просканирует проект и сгенерирует первую версию CLAUDE.md. Дальше — дополняй по ходу работы, когда замечаешь что агент ошибается. Остальное добавится само.
